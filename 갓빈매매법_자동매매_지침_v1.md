# 갓빈이 매매법 자동매매 구현 지침 (v1)
## Binance USDⓈ-M Futures · Python

> 이 문서는 DCinside 전자화폐/차트 갤러리의 "갓빈이 매매법"(원본, 김블루 정리)과 제주도트레이더의 응용·Q&A 글 3개를 종합해, **Claude Code가 모호함 없이 코드로 구현**할 수 있도록 전략을 결정론적 규칙으로 옮긴 지침이다.
>
> **표기 규칙**
> - `[기본값]` : 작성자가 합리적으로 정한 기본 수치. 사용자가 자유롭게 변경 가능.
> - `[원글값]` : 원문 글쓴이가 명시한 값을 그대로 옮긴 것.
> - `[사용자확인]` : 사용자가 직접 정해야 의미가 또렷해지는 항목. 문서 끝 §13에 모아둠.
> - 모든 파라미터는 §11 표 **한 곳**에서 관리한다. 로직 코드에 숫자를 하드코딩하지 말 것.

## 0. 목적과 범위
- **목적**: 15분봉 기준 "추세추종 + 역추세 되돌림 진입" 스캘핑 전략을 자동매매로 구현.
- **범위**: 단일 거래소(Binance USDⓈ-M 무기한 선물), 단일 계정, 동시 1포지션을 전제로 한 v1.
- **검증 순서 강제**: 페이퍼/테스트넷 → 백테스트 검증 → (그 다음에만) 실자금. 전략 특성상 고배율이라 손실 누적 속도가 매우 빠르며, 원글 작성자도 "확실한 자리에서 들어가도 80%는 손절각"을 언급했다.

## 1. 전제 / 실행 환경
- **거래소**: Binance USDⓈ-M Futures (USDT 마진). 시드 단위는 USDT. (원글은 KRW 기준 → §7에서 환산 규칙 명시)
- **언어/스택(가정)**: Python 3.10+. 핵심 라이브러리는 §12 참고.
- **타임프레임**: 진입 판정 = 15m(주). 추세 판정 = 1h(주), 4h(보조 확인).
- **데이터**: 실시간 kline(웹소켓) + 보조 REST. 모든 지표는 **확정된 봉**(closed candle) 기준으로 계산.

## 2. 핵심 개념 정의 (★ 가장 중요)
원문의 '느낌' 표현을 전부 수치/조건으로 고정한다. 이 섹션이 전략의 본체다.

### 2-1. 캔들 구성요소 (한 봉의 O/H/L/C)
- `body`(몸통) = `abs(close − open)`
- `range`(전체 길이) = `high − low`  → **0이면 그 봉의 신호는 무효 처리**(0 나눗셈 방지)
- `upper_wick`(윗꼬리) = `high − max(open, close)`
- `lower_wick`(아랫꼬리) = `min(open, close) − low`

### 2-2. 기본 판정
- 양봉 = `close > open` / 음봉 = `close < open`
- 도지 = `body / range ≤ 0.10` `[기본값]`
- `avg_body(N)` = 직전 N개 확정봉 body의 평균. N = 20 `[기본값]`
- `ATR` = ATR(14) (진입 타임프레임 기준) `[기본값 기간 14]`

### 2-3. 전략 핵심 용어
- **장대봉(impulse candle)**: `body ≥ avg_body(20) × 1.5` `[기본값]` (권장: `body ≥ max(avg_body(20)×1.5, ATR×1.0)`). 추세에 **역행**하는 방향의 장대봉(롱이면 장대 음봉, 숏이면 장대 양봉)이 "강한 자리"를 만든다.
- **되돌림 꼬리(rejection wick)**: 롱 기준 `lower_wick / range ≥ 0.33` `[기본값]` **그리고** `lower_wick ≥ body × 1.0` `[기본값]`. (숏은 `upper_wick`로 동일)
- **양전(bullish flip, 롱)**: `(close > open) AND (close > prev_close)`. 숏의 음전 = `(close < open) AND (close < prev_close)`.
- **확인봉 최소 크기**: 신호 확정 직후 형성되는 캔들 `body ≥ avg_body(20) × 0.5` `[기본값]`. 이보다 작으면 모멘텀 부족으로 간주(원문 "3~5분 내 적당한 양봉 안 나오면 버려라").

> 이 임계값들이 전략의 민감도를 좌우한다. 반드시 백테스트로 조정할 것.

## 3. 추세 판정 (Trend Filter)
- **지표**: EMA200 (1h). 보조: EMA200 (4h).
- **방향**:
  - `UP`   = (1h close > 1h EMA200) **AND** (4h close > 4h EMA200)
  - `DOWN` = (1h close < 1h EMA200) **AND** (4h close < 4h EMA200)
  - 그 외(1h/4h 불일치) = `NO_TRADE`
- **알트 정렬**: 알트 진입 방향은 반드시 **BTC의 1h 추세 방향과 동일**해야 한다. (BTC `UP`일 때만 알트 롱, `DOWN`일 때만 알트 숏)
- **횡보 배제 게이트**(전략이 횡보에 취약): `ADX(14, 1h) ≥ 20` `[기본값]`. 미만이면 해당 종목 `NO_TRADE`.

## 4. 진입 로직 (Entry)
**진입은 추세 방향으로만.** 아래는 롱 기준, 숏은 모든 조건의 거울상.

### 4-1. 진입 모드 `[사용자확인]`
- **Mode A — 종가 확정 (v1 권장/기본)**: 15m 봉이 **확정 종료**된 시점에 직전 확정봉이 진입 조건을 만족하면, 다음 봉 시가 부근에 지정가 진입. 결정론적이라 백테스트·디버깅이 쉽다. 단, 원글의 '봉 진행 중 즉시 진입'보다 한 박자 늦다.
- **Mode B — 봉 진행 중 (원문 충실)**: 형성 중인 봉을 모니터링하다 §4-2 조건이 충족되는 순간 진입. 원문과 가장 가깝지만 노이즈/리페인팅 위험이 커 구현 난도가 높다.
- 기본값: **Mode A**.

### 4-2. 진입 조건 (모두 충족 시 신호 발생, 롱)
1. `trend == UP` 그리고 (알트면) `BTC 1h 추세 == UP`
2. `ADX(14,1h) ≥ 20`
3. 신호봉(Mode A=직전 확정봉, Mode B=형성봉)이:
   - a. **양전**: `close > open` AND `close > prev_close`
   - b. **되돌림 꼬리**: `lower_wick/range ≥ 0.33` AND `lower_wick ≥ body`
   - c. (품질 등급, 선택) 직전봉이 '추세 역행 장대 음봉'이면 **등급 A**, 아니면 **등급 B**.
4. (Mode B 한정) 봉 시작 후 `T_CONFIRM = 5분` `[기본값]` 이내에 위 조건 성립해야 함. 미성립 시 이번 봉 스킵.

### 4-3. 주문 방식
- **진입 = 지정가(LIMIT)**. 진입가 = 신호 기준가에서 유리한 방향으로 `ENTRY_TICK_OFFSET = 1틱` `[기본값, 1~2틱]` 떨어진 가격(롱이면 살짝 아래). 가격·수량은 거래소 `tickSize`/`stepSize`로 반올림.
- 원글(김블루)은 전량 **시장가**지만, 응용글(제주)대로 고배율에서는 지정가가 평단상 유리하므로 **지정가를 기본**으로 한다.
- **미체결 처리**: 진입 지정가가 `ENTRY_FILL_TIMEOUT`(Mode A=다음 봉 종료까지 / Mode B=신호 후 90초) `[기본값]` 내 미체결이면 **취소(추격 금지)** 후 다음 유효 봉 대기.

## 5. 손절 (Stop Loss)
- **즉시 설정**: 진입 체결 직후 `reduce-only STOP_MARKET` 주문을 곧바로 건다(체결~손절등록 공백 최소화).
- **가격 기준**: `workingType = CONTRACT_PRICE`(=Last Price). **MARK_PRICE 사용 금지** — 급등락 시 미체결 참사 방지(원문 강조).
- **손절선(롱)**:
  1. **기본**: 신호봉 **직전봉의 중간값** `mid = (prev_high + prev_low)/2` 이 깨지는 지점. (장대봉의 중앙값이 지지/저항 역할 — 원문 근거)
  2. 직전봉 body가 `avg_body(20)×0.5` 미만으로 **너무 작으면** → **진입봉(신호봉)의 저점(low)** 에 손절.
  3. **안전장치**: 위 SL이 진입가보다 위(롱인데 손절이 진입가 위)거나 손절폭이 진입가의 0.1% 미만이면 → 진입봉 저점으로 폴백. 그래도 모호하면 **진입 취소**.
- **R(리스크) 정의**: `R = entry_price − SL_price` (롱). 포지션 사이징·익절의 기준값.

---

## 6. 익절 / 청산 (Take Profit / Exit)
세 가지 청산 트리거 중 **먼저 닿는 것**으로 청산한다.
1. **SL 도달** → 손절 청산.
2. **TP 도달** → 익절 청산. `TP(롱) = entry_price + R × RR_RATIO`, `RR_RATIO = 1.3` `[기본값, 원문 1:1.3]`.
3. **봉마감 청산**: 진입한 15m 봉이 종료되는 시점에 1·2가 모두 미발생이면 종가에 청산.
   - **예외(불타기 보유)**: 청산 시점 미실현 수익률 ≥ `PROFIT_HOLD_PCT = 10%` `[기본값, 원문]` 이면 청산하지 않고 다음 봉으로 보유 연장. 이후 매 봉 종료마다 동일 규칙 재평가(트레일링처럼 동작). 보유 연장 중 SL은 본전(entry) 또는 직전 봉 저점으로 상향 `[사용자확인]`.
- **청산 모드 토글** `[사용자확인]`: `EXIT_MODE ∈ {RR_ONLY, CANDLE_ONLY, HYBRID}`. 기본 = **HYBRID**(1·2·3 모두 적용).

## 7. 포지션 / 리스크 관리

### 7-1. 레버리지 (원글 표 — 참고용)
| 시드(KRW) | 비메이저 알트 | 메이저 알트 |
|---|---|---|
| 20만 ~ 100만 | 20x | 45x |
| 100만 ~ 1000만 | 5 ~ 10x | 20 ~ 30x |

`[원글값]`. **이 수치는 매우 공격적이다.** Binance는 USDT 기준이므로 `USDT_KRW` 환율 `[사용자확인]`(예: 1,400)로 환산해 구간 적용.

### 7-2. 포지션 사이징 (권장 방식으로 재정의)
- `SIZING_MODE` `[사용자확인]` ∈ `{RISK_PCT(권장), FIXED_NOTIONAL(원문식)}`
- **RISK_PCT (기본)**: SL 도달 시 손실이 자본의 `RISK_PER_TRADE = 1%` `[기본값, 보수적]` 이 되도록 수량 산출.
  - `qty = (equity × RISK_PER_TRADE) / R` (R은 §5 가격 손절폭)
  - 레버리지는 "결과로 따라오는 값"이 되고, `LEVERAGE_CAP`로 상한을 건다.
- **FIXED_NOTIONAL (원문식)**: 진입당 명목금액 고정(원글 "100만원씩"). `NOTIONAL_PER_TRADE` `[사용자확인]`.
- `LEVERAGE_CAP = 10x` `[기본값, 보수적]` (원글 표 20~45x를 쓰려면 사용자가 의식적으로 상향). 고배율일수록 청산 위험이 급증한다.

### 7-3. 거래 빈도 / 추격 / FOMO
- 동시 보유 포지션 = 1 `[기본값]`
- 15m 봉당 신규 진입 = 최대 1회 `[기본값]`
- **추격매수 금지**: 손절 후 같은 봉/같은 셋업 재진입 금지, 다음 유효 봉까지 대기(원문 "추격매수 안 함").
- **FOMO 잠금**: 신호를 놓쳤거나 미체결 취소된 경우 봉 진행 중 재시도 금지. 진입 평가는 **오직 새 봉 평가 시점**에만 발생(원문 "포모 반드시 참아라 — 타점은 매 15분 다시 온다").

## 9. 실행 루프 / 상태 머신
종목별로 아래 상태를 독립 관리하되, **전역 제약(동시 1포지션)** 을 우선한다.

```
states: SCANNING → SETUP_DETECTED → ENTRY_PENDING → IN_POSITION → EXIT_PENDING → COOLDOWN → SCANNING

SCANNING:
  매 15m 봉 종료(Mode A) 또는 틱(Mode B)마다, §8 필터 통과 종목에 대해 §4-2 평가.
  신호 발생 + 전역 포지션 없음 → SETUP_DETECTED

SETUP_DETECTED:
  §5로 SL 후보 계산 → R, 안전장치 검증.
  유효 → §4-3 지정가 진입 주문 → ENTRY_PENDING
  무효(R 비정상 등) → SCANNING

ENTRY_PENDING:
  체결 → 즉시 §5 STOP_MARKET(reduce-only, Last Price)
        + §6 TP LIMIT(reduce-only) 등록 → IN_POSITION
  ENTRY_FILL_TIMEOUT 초과 미체결 → 주문 취소 → COOLDOWN

IN_POSITION:
  SL/TP 체결 감시 + 진입봉 종료 감시(§6 봉마감 청산, 10% 예외).
  청산 트리거 발생 → EXIT_PENDING

EXIT_PENDING:
  잔여 reduce-only 주문 취소 후 청산 확인 → COOLDOWN

COOLDOWN:
  최소 COOLDOWN_BARS = 1봉 대기(추격/FOMO 방지) → SCANNING
```

## 10. 안전장치 (필수)
- **일일 손실 한도(킬 스위치)**: 당일 누적 손실이 `MAX_DAILY_LOSS_PCT = 5%` `[기본값, 보수적]` 도달 시 당일 신규 진입 전면 중지.
- **연속 손절 차단**: 연속 `N_CONSEC_SL = 3` `[기본값]` 손절 시 `COOLDOWN_LONG = 1시간` `[기본값]` 휴식.
- 진입 직후 **SL 등록 실패** 시 → 즉시 포지션 시장가 청산(무방비 포지션 금지).
- 웹소켓 단절/예외 → 재연결, 재연결 전까지 신규 진입 금지. 보유 포지션의 SL은 **거래소 측에 이미 올라가 있어야** 함(로컬 메모리 의존 금지).
- 모든 주문 전 가격/수량을 `tickSize`/`stepSize`/`minNotional`에 맞게 검증·반올림.
- 모든 의사결정·주문·체결을 **구조화 로그**로 남김(백테스트 재현용).

- 시간 동기화 / 레이트리밋 / 서명 처리 준수.

## 13. 사용자 확인/결정 필요 항목 (정리)
아래만 정해주면 지침이 완전해진다. 미정 시 위 기본값으로 진행한다.
1. **ENTRY_MODE**: A(종가확정·권장) vs B(봉중·원문충실)
2. **EXIT_MODE**: HYBRID(기본) vs RR_ONLY vs CANDLE_ONLY
3. **SIZING_MODE**: RISK_PCT(권장) vs FIXED_NOTIONAL(원문 "진입당 고정금액")
4. **RISK_PER_TRADE %**(기본 1%) / **LEVERAGE_CAP**(기본 10x) — 원글 표(20~45x)를 쓸지 여부
5. **신호 등급**: A·B 모두 vs '추세 역행 장대봉 직후' 등급 A만
6. **PROFIT_HOLD_PCT 기준**: 가격 변동률 vs 레버리지 반영 ROE
7. **메이저/비메이저 코인 분류 목록** (레버리지·필터용)
8. **BTC 자체 매매 포함 여부**
9. **USDT_KRW 환율**(레버리지 표 환산용) / **시드 규모**
10. **MAX_DAILY_LOSS_PCT** 등 안전장치 수치 조정 여부
