# 갓빈 매매법 페이퍼 트레이딩 작업내역

## 2026-06-08 (초기 구축)

### 프로젝트 개요
DC인사이드 커뮤니티에서 공유된 "갓빈 매매법"을 자동 탐지하는 페이퍼 트레이딩 봇.
Binance Futures 상위 10종목을 15분봉 기준으로 스캔하고, 신호 발생 시 Notion 매매일지에 자동 기록한다.
향후 자동매매로 전환을 전제로 구조 설계.

---

### 전략 규칙 (갓빈 매매법)

| 항목 | 내용 |
|------|------|
| 추세 판단 | 1H + 4H 200 EMA 기준 동일 방향일 때만 진입 |
| 대상 종목 | Binance Futures 24h 거래대금 상위 10개 |
| 진입 타이밍 | 15분봉 기준 |
| 롱 신호 | 직전 음봉(적당한 몸통) + 신호봉이 아래꼬리 그리며 양봉 마감 |
| 숏 신호 | 직전 양봉(적당한 몸통) + 신호봉이 위꼬리 그리며 음봉 마감 |
| 손절 | 신호봉 저가(롱) / 신호봉 고가(숏) |
| 익절 | 리스크 × 1.3 (RR 1:1.3) |
| 강제 청산 | 15분봉 마감 시 SL/TP 미도달이면 종가 청산 |

### 완료 작업

#### 1. 프로젝트 구조 생성 (`C:\Users\user\claude-workspace\godbin`)

| 파일 | 역할 |
|------|------|
| `config.py` | 전략 파라미터 (EMA 기간, RR비율, 포지션 크기 등) |
| `exchange.py` | Binance 공개 API 래퍼 (상위 10종목 조회, OHLCV 수신) |
| `strategy.py` | 200 EMA 추세 판단 + 15분봉 진입 신호 탐지 |
| `paper_trade.py` | 가상 포지션 관리, SL/TP/강제청산 처리, PnL 계산 |
| `notion_logger.py` | Notion API 연동 (진입 등록 / 청산 업데이트) |
| `main.py` | 메인 루프 (15분봉 마감 시마다 자동 스캔) |
| `run.bat` | 메인 루프 실행 단축 |
| `scan.bat` | 현재 신호 즉시 확인 |
| `run_forever.bat` | 크래시 시 10초 후 자동 재시작 루프 |

#### 2. 페이퍼 트레이딩 로직

- 15분봉 마감(:00 :15 :30 :45 UTC)마다 상위 10종목 전체 스캔
- 신호 발생 → 신호봉 종가로 가상 진입, `trades.json`에 저장
- 다음 봉 마감 시 OHLCV 고저 기준으로 SL/TP 도달 여부 판정 후 청산
- 동일 종목 중복 진입 방지

### 내일 할 일

#### Google Cloud Free Tier VM 세팅 (24/7 자동 실행 목적)

1. [console.cloud.google.com](https://console.cloud.google.com) 접속 → Gmail 로그인
2. 새 프로젝트 생성
3. Compute Engine → VM 인스턴스 만들기
   - 리전: `us-central1` (영구 무료 대상)
   - 머신 유형: `e2-micro`
   - OS: Ubuntu 22.04 LTS
4. VM SSH 접속 후 실행:
   ```bash
   git clone https://github.com/TESTER-KOR/godbin.git
   cd godbin
   echo "노션토큰" > notion_token.txt
   bash deploy/setup.sh
   ```
5. 실행 확인:
   ```bash
   sudo systemctl status godbin
   journalctl -u godbin -f
   ```

> 주의: 신용카드 등록 필요하지만 e2-micro는 영구 무료 (자동 업그레이드 안 하면 청구 없음)
