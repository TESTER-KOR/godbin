# 갓빈 매매법 페이퍼 트레이딩 작업내역

## 2026-06-08 (초기 구축)

### 프로젝트 개요
DC인사이드 커뮤니티에서 공유된 "갓빈 매매법"을 자동 탐지하는 페이퍼 트레이딩 봇.
Binance Futures 상위 10종목을 15분봉 기준으로 스캔하고, 신호 발생 시 Notion 매매일지에 자동 기록한다.
향후 자동매매로 전환을 전제로 구조 설계.

---

### 전략 규칙 (갓빈 매매법)

| 항목 | 내용 |
|------|------|
| 추세 판단 | 1H + 4H 200 EMA 기준 동일 방향일 때만 진입 |
| 대상 종목 | Binance Futures 24h 거래대금 상위 10개 |
| 진입 타이밍 | 15분봉 기준 |
| 롱 신호 | 직전 음봉(적당한 몸통) + 신호봉이 아래꼬리 그리며 양봉 마감 |
| 숏 신호 | 직전 양봉(적당한 몸통) + 신호봉이 위꼬리 그리며 음봉 마감 |
| 손절 | 신호봉 저가(롱) / 신호봉 고가(숏) |
| 익절 | 리스크 × 1.3 (RR 1:1.3) |
| 강제 청산 | 15분봉 마감 시 SL/TP 미도달이면 종가 청산 |

#### 1. 손절 로직 변경 (§5)

| | 이전 | 변경 후 |
|---|---|---|
| 기본 SL (롱) | 신호봉 저가 | **직전봉 중간값** `(prev_high + prev_low) / 2` |
| 기본 SL (숏) | 신호봉 고가 | **직전봉 중간값** |
| 폴백 조건 | 없음 | 직전봉 body < avg_body × 0.5 이면 신호봉 저/고점 사용 |
| 안전장치 | 없음 | SL이 진입가 반대편이거나 0.1% 미만이면 폴백 후 재검증, 여전히 무효면 진입 취소 |

원문 근거: "장대봉의 중앙값이 지지/저항 역할" (원문 §5)

---

#### 2. 진입 신호 조건 강화 (§4-2)

| 조건 | 이전 | 변경 후 |
|---|---|---|
| 신호봉 방향 | 양봉 여부만 | **양전(bullish flip)**: `close > open` AND `close > prev_close` |
| 되돌림 꼬리 | `꼬리/몸통 ≥ 0.25` | `꼬리/전체범위 ≥ 0.33` **AND** `꼬리/몸통 ≥ 1.0` |
| 도지 필터 | 없음 | `body/range ≤ 0.10` 이면 신호 무효 |
| 신호 등급 | 없음 | 직전봉이 추세 역행 장대봉이면 **등급 A**, 아니면 **등급 B** |

#### 3. ADX 횡보 배제 필터 추가 (§3)

- `ADX(14, 1h) < 20` 이면 해당 종목 `NO_TRADE`
- `indicators.py`에 Wilder 방식 ADX 직접 구현 (외부 라이브러리 불필요)

---

#### 4. BTC 추세 정렬 추가 (§3)

- 알트코인은 **BTC 1h 추세 방향과 동일한 경우에만** 진입
- BTC 자체는 정렬 체크 제외
- BTC가 `NO_TRADE`이면 전체 스캔 스킵

---

#### 5. 포지션 사이징 변경 (§7-2)

| | 이전 | 변경 후 |
|---|---|---|
| 방식 | 고정 200 USDT | **RISK_PCT**: SL 도달 시 손실 = 자본의 1% |
| 레버리지 | 고정 없음 | 자동 산출, LEVERAGE_CAP(10x) 초과 시 사이즈 축소 |
| 계산 | 없음 | `size_usdt = (capital × 0.01) / R × entry` |

---

#### 6. 불타기 보유 로직 추가 (§6)

- EXIT_MODE = `HYBRID` (기본)
- SL/TP 미도달 + **미실현 수익 ≥ 10%** 이면 봉마감 청산 없이 다음 봉으로 보유 연장
- 보유 연장 시 SL을 본전(entry)으로 상향

---

#### 7. 안전장치 추가 (§10)

| 항목 | 내용 |
|------|------|
| 일일 손실 한도 | 당일 누적 손실 ≥ 자본의 5% → 당일 신규 진입 전면 중단 (킬스위치) |
| 연속 손절 차단 | 연속 3회 손절 시 1시간 쿨다운 |
| 상태 저장 | `trades.json`에 daily_pnl / consec_sl / cooldown_until 영속화 |
